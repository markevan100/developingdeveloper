Handy Terminal Commands

The end? Oh no, it's only the beginning.


March 2019

'Terminal', forming, or situated at the end or extremity of something. The terminal on your computer is anything but 'the end'. It's a whole new beginning to how a human can interact with a machine.

Perhaps I have yet to grasp the exact reason behind using the terminal over navigating my Mac computer through its beautifully designed interface, but it does seem once you learn how to move around the terminal, that navigation becomes quick, utilitarian, and 'simple'. I say 'simple' because the terminal does have its own language that appears to require lots of memorization to become familiar with. This seems to be the method by which most programmers move about their machines... I am going to do some more follow up information as to exactly why this is.

While reading 'A Mind for Numbers', to help restructure my cognitive process to be more effecient at learning coding, the author reccamended the best way to retain new information is to not let more than 24 hrs go by without reviewing that information. Since I have found the terminal quite foriegn, I am going to attempt to interact with the terminal atleast once a day in hope the commands and method of moving around my computer soon become second nature.

I've jumped into a fun project with some friends of mine creating an app. Dont' worry, I'll dive into the app in another post. I have been learning how to use the terminal to move around the app project and to push and pull files from github. Below I share a list of the initial commands I've been using.

Basic Commands

  • ls - lists the files in the current working directory
  • cd - change directory
  • atom . - open where I am in atom
  • [command] t - open another terminal

How to Move Files Around and Check Status

Knowing where exactly you are located in the terminal and how to move around is very important.

  • git status - shows if and what files have been changed
  • git checkout - change one branch to the other
  • git checkout -b [name] - create (build) a new branch, should always be done after a push
  • git branch -d [name] - delete a branch, keep files clean and delete branches you don't need
  • git stash - ??

Get Files to the Cloud

The process to send a file to the cloud follows this order... add then commit then push.

  • git add . - add all files where I am currently located in directory to be ready to commit
  • git add [name of file] - add that exact file to be ready to commit
  • git commit -m "[message]" - commits file to be pushed up into the cloud, message is mandatory and should be addressed to the person(s) you are pushing the file too with some detail about what you are pushing, often times curse words exist in these messages and this handy little program captures all of them, haha
  • git push - pushes the file to the cloud, if it fails, just copy the text it prompts you to use and hit enter
  • git pull - pulls the file down from the cloud

Server Commands

  • rails s - starts server
  • control c - leave server
